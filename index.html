Random Geometric Graphs

Visualization of random geometric graphs using Common Lisp.

The implementation is split into two parts.

  1. polar.lisp uses the Common Lisp Object System (CLOS) to define a class of polar coordinates with some basic operations like translation between polar and rectangular coordinate systems, adding coordinates etc…
  2. rgg.lisp implements random geometric graphs of verticies moving inside of a field. Using the run-to method the simulation may be visualized using a gnuplot front-end. Thanks to the lisp REPL it is possible to update simulation parameters during a running simulation. The important parameters are listed below. 
    (defvar *field-radius* 0.5)            ; size of the field
(defvar *r* 0.1)                       ; connectivity radius
(defvar *s* 0.01)                      ; step length (speed)
(defvar *m* 8)                         ; expected steps between change
(defvar *vertices* '())                ; holds the vertices

The following steps may be used to run this locally.

  1. Make a fifo through which CL can dump graph directives to gnuplot 
    mkfifo /tmp/feedgnuplot
gnuplot < /tmp/feedgnuplot
  2. Make an initial population of vertices 
    (populate 20)
  3. Start a simulation running in another thread 
    (setq *gnuplot* (run-to #P"/tmp/feedgnuplot"))
  4. Change some simulation parameters from the REPL and see their effects in the gnuplot window. 
    ;; increase edge radius
(setf *r* 0.2)
;; increase vertex speed
(setf *s* 0.05)
;; increase the number of vertices to 80
(populate 80)
  5. When finished terminate the simulation thread 
    (terminate-thread *gnuplot*)
